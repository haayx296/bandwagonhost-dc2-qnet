## DC2 QNET 是什么机房？线路是什么情况？

DC2 QNET，机房编号 USCA\_2，坐落在洛杉矶，目前已经迁移到了 Coresite LA2 数据中心（原来在 Quadranet，因为那边老停电所以搬了）。顺带一提，搬瓦工官方现在把这个机房改名叫 **DC2 AO**，AO 是 Asia Optimized 的缩写。名字换了，机房编号没变，USCA\_2 还是那个机房。

线路这块，最容易让人困惑的是：DC2 QNET 到底走不走 CN2？

答案是：**走，但不是搬瓦工官方保证的**。这个机房本身（Coresite/原 Quadranet）在网络层面接入了 CN2 线路，电信用户回程会走一部分 CN2 GT，但搬瓦工官方并不将这个机房定义为 CN2 机房，所以如果你用的是 KVM 套餐迁到 DC2，流量不会像迁到 DC3 那样打三折——因为官方压根没把它算进 CN2 套餐。

总结一下线路情况：

- 电信：CN2 GT（机房层面接入，非官方保证）
- 移动：直连
- 联通：直连
- 延迟：电信平均在 170ms 左右，三网都在 200ms 以内

## 速度和延迟表现实际怎样？

三网延迟都不高，170 到 200ms 这个区间对洛杉矶来说算正常水平。下载速度全球大多数方向都不错，到中国大陆的速度白天还可以，晚高峰的时候会有所下降。

有一点要说清楚：**丢包率这个机房历史上偶有波动**。平峰时段用下来问题不大，但如果你需要 7\*24 小时稳定跑实时业务，这个机房不是最优选。

好消息是，搬瓦工 2024 年完成了从 Quadranet 到 Coresite LA2 的机房迁移，新机房升级了网络端口和供电，稳定性比以前好了不少，之前那些停电导致频繁宕机的情况基本消失了。

硬盘读写速度在迁移后的表现相当不错，这点是实测有感知的提升。
